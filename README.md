Энэхүү төсөл нь [Gym Management System] системийн өгөгдлийн санг зохион байгуулсан болно.

Систем нь дараах үндсэн хүснэгтүүдээс бүрдэнэ:

### 4-р хэсэг:Онол (Security)
4-р хэсэг: Онол (Security) — Хариултууд
Энэ хэсэгт өгөгдлийн сангийн аюулгүй байдал болон бүрэн бүтэн байдалтай холбоотой онолын асуултуудад дараах байдлаар хариулж бичээрэй:
1. CHECK constraint гэж юу вэ? Ямар нөхцөлд ашигладаг вэ? Жишээ өг.
* Тодорхойлолт: CHECK хязгаарлалт нь хүснэгтийн баганад орж буй өгөгдөл тодорхой нэг логик нөхцөлийг хангаж байгаа эсэхийг шалгахад ашиглагддаг. Хэрэв оруулж буй өгөгдөл уг нөхцөлийг хангахгүй бол өгөгдлийн сан уг үйлдлийг цуцалдаг.
* Ашиглах нөхцөл: Тоон утгыг тодорхой интервалд барих (жишээ нь: нас 0-ээс их байх), эсвэл зөвхөн зөвшөөрөгдсөн утгуудыг оруулах үед ашиглана.
* Жишээ: ```sql
CREATE TABLE members (
age INT,
CHECK (age >= 16) -- Гишүүн заавал 16-аас дээш настай байх ёстой
);


2. DISTINCT түлхүүр үг ямар үүрэгтэй вэ? Яагаад зарим үед хэрэгтэй байдаг вэ?
* Үүрэг: SELECT асуулгын үр дүнгээс давхардсан мөрүүдийг устгаж, зөвхөн ялгаатай (unique) утгуудыг харуулдаг.
* Хэрэгцээ: Өгөгдлийн санд нэг мэдээлэл олон удаа бүртгэгдсэн байж болно (жишээ нь: нэг гишүүн олон удаа ирц бүртгүүлсэн). Хэрэв бидэнд "Нийт хэдэн хүн ирсэн бэ?" биш, харин "Яг ямар ямар хүмүүс ирсэн бэ?" гэдэг жагсаалт хэрэгтэй бол DISTINCT-ийг ашиглан нэрийг нь давхардуулахгүйгээр хардаг.
* Жишээ: SELECT DISTINCT member_id FROM attendance;
3. Өгөгдлийн сангийн нэвтрэх эрхийг (Authentication) зөв тохируулахгүй бол ямар эрсдэл үүсэж болох вэ?
Нэвтрэх эрхийг буруу тохируулснаар дараах эрсдэлүүд үүснэ:
* Өгөгдөл алдагдах (Data Breach): Зөвшөөрөлгүй хэрэглэгч байгууллагын нууц мэдээлэл, үйлчлүүлэгчдийн хувийн мэдээллийг хулгайлах.
* Өгөгдөл устгагдах эсвэл өөрчлөгдөх: Гэмт этгээд эсвэл туршлагагүй хэрэглэгч чухал хүснэгтүүдийг устгах (DROP), эсвэл мэдээллийг хуурамчаар өөрчлөх.
* SQL Injection халдлага: Нэвтрэх эрх сул байх нь хакеруудад систем рүү хортой код илгээх боломжийг олгодог.
* Системийн зогсолт: Хэт их эрхтэй хэрэглэгч системийн тохиргоог алдагдуулснаар өгөгдлийн сан ажиллахгүй болох эрсдэлтэй.
